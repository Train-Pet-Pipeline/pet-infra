## Weaknesses（按结构性分类，按严重度排）

### 1. orchestrator → plugin handoff 静默断裂（5 处）

F009 / F011 / F012 / F014 / F016 都是同款问题：声称的 plugin 接口落地时**没真实跑过**就发布了。

| Finding | 现象 |
|---|---|
| F009 | `pet run` 不 auto-discover plugins |
| F011 | LF wrapper 用 private API 不是 public |
| F012 | `pet run` 不持久 ModelCard |
| F014 | 3 个 fusion plugin 缺 concrete run() 根本不能被 orchestrator 调用 |
| F016 | ClearML `fallback_null` 不抓 MissingConfigError，policy 文档 是 lie |

**根因**：cross-repo-smoke-install.yml 只验装包，**不端到端跑**。

### 2. schema-consumer contract gap（F001）

pet-schema v3.2.0 silently 删了 ShareGPTSFTSample.images，没人发现，因为 schema unit test 只验自己的 class，没有 consumer contract test。Phase 3A 训练用了这个字段，eco-cleanup 期间被悄悄破坏。

### 3. audio modality 实际能力 < 文档（F008 + F017）

- **F008**：custom MobileNetV2 与官方 PANNs checkpoint 通道数不匹配，从来没真正能加载官方权重 — v2.2.0 用 panns_inference 包重写 plugin 修了
- **F017**：5-class taxonomy（eating/drinking/vomiting/ambient/other）是给**喂食器行为**设计的，不是物种 ID。我们用 cat-meow / dog-bark 测试是错配数据。**audio 真实能力其实没在 designed domain 验证过**

### 4. video path 完全未测

169 video frames 提取出来后从未 annotate / train / eval。**几乎肯定有 F001/F005-class 同款回归**，因为 image path 的 cross-repo 契约 bug 在 video path 没理由不复刻。

### 5. replay-determinism 未端到端验证

F012 fix 让 ModelCard 持久化了，但**实际的 replay-across-commits 从未跑过**。Comparability 维度的核心承诺只验到一半。

### 6. report 文档诚实漏洞（流程 bug）

B1.7 编造了 "dog 0.87/cat 0.91" — 是 living doc 写多了把"打算做"和"已做"混淆。靠用户两次"怎么样了"meta-question 才挖出来。**这是流程 bug，不是单次 bug**。
