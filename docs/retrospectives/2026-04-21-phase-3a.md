## 5. North Star §0.2.1 自检（DEBT-4）

### 可插拔性（Pluggability）: **5 / 5**
证据:
- `pet_train.plugins._register` + `pet_eval.plugins._register` hooked via `pet_infra.plugins` entry-point group — orchestrator discovers plugins zero-config.
- Adding a new trainer = register one class + ship package (no core changes): see `LlamaFactoryDPOTrainer` added in PR #16 without touching pet-infra.
- AudioEvaluator lazily imports `pet_train.audio.inference` — proves cross-repo plugin chain works without central wiring.

### 灵活性（Flexibility）: **4 / 5**
证据:
- `params.yaml` holds all numeric thresholds (gate/smoke/eval namespaces).
- Stage configs in `configs/smoke/*.yaml` are pure data; plugin class only reads the dict.
- OmegaConf overrides work (`pet run recipe.yaml -m train.lr=1e-4,3e-4` path exists via Hydra launcher guard P0-F).
- Deduction: full Hydra defaults-list inheritance is deferred to Phase 3B+ (compose.py docstring acknowledges).

### 可扩展性（Extensibility）: **4 / 5**
证据:
- 6 registries live in `pet_infra.registry` (TRAINERS/EVALUATORS/CONVERTERS/METRICS/DATASETS/STORAGE). Adding a CONVERTERS member (Phase 3B pet-quantize) requires no orchestrator change — `stage_executor.STAGE_REGISTRIES` already wires `converters`.
- ExperimentLogger ABC (P0-B) + factory (P0-C) supports null/offline/saas/self_hosted modes; adding a new logger = new subclass.
- Deduction: Hydra launcher guard is "basic only" — multirun DAG semantics deferred.

### 可对比性（Comparability）: **5 / 5**
证据:
- Every stage output is `ModelCard` with consistent fields (`id`, `gate_status`, `metrics` map, `lineage_role`).
- `precompute_card_id(recipe_id, stage, config_sha)` gives deterministic IDs — cache hits across runs are provable by hash.
- gate_status in {passed/failed} enables threshold comparison at the recipe level regardless of plugin class.

**最低维度 4 ≥ 3，Phase 3A 通过。**
